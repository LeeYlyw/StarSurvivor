# 1. 게임 소개

## 게임 개요

**Space Survival**은 Android CustomView 기반의 2D 생존형 슈팅 게임이다.

플레이어는 우주선을 조작하여 화면 안을 자유롭게 이동하고, 자동으로 발사되는 총알로 적 비행선을 파괴한다. 적 비행선은 화면 가장자리에서 등장하여 플레이어를 향해 접근하며, 운석은 총알로 파괴할 수 없는 장애물로 등장한다.

플레이어는 적과 운석을 피하면서 최대한 오래 생존하고 높은 점수를 얻는 것을 목표로 한다.

본 프로젝트는 수업에서 사용한 a2dg 프레임워크를 기반으로 제작하였다.

## 기존 게임과의 차이점

좌우 고정 이동을 탈피한 전방향 맵 활용

기존의 종스크롤 슈팅 게임들은 플레이어 기체가 화면 하단 라인에 고정되어 좌우로만 움직이는 제한적인 조작 방식을 가졌습니다.

반면 본 게임은 플레이어가 화면 전체, 전방향을 자유롭게 누비며 이동할 수 있도록 설계하여 조작의 자유도를 극대화하고 맵 전체를 넓게 활용하도록 했습니다.

손가락 가림을 방지하는 Y축 터치 오프셋 조작

전방향으로 움직일 때 터치한 자리에 기체가 그대로 있으면 손가락에 화면이 가려져 적들을 피하기 어렵습니다.

이를 해결하기 위해 실제 터치한 위치보다 항상 150픽셀 위에서 우주선이 정밀하게 따라오도록 조작 오프셋을 적용해 넓은 시야를 확보했습니다.

## 게임 목표

적과 운석을 피하면서 최대한 오래 생존하고 높은 점수를 획득하는 것을 목표로 한다.

# 6. 사용된 기술

## 개발 환경

* Android Studio
* Git
* GitHub

## 언어

* Kotlin

## 기술 추가

프레임워크 기본 구조: BaseGameActivity, GameView, GameContext를 모듈로 연결하여 전체 게임 루프 작동에 활용

오브젝트 계층 상속: Sprite 클래스를 상속받아 게임 내 객체(Player, Bullet, Enemy, Asteroid) 구현

레이어 기반 관리: World 객체를 사용하여 각 오브젝트를 레이어 단위(Layer.BULLET, Layer.ENEMY 등)로 분할 관리

안전한 메모리 수거: 객체 충돌 및 화면 이탈 시, 데이터 인덱스가 깨지지 않도록 역순으로 순회하며 안전하게 소거하는 forEachReversedAt 루프 구조 활용

효과음 재생: SoundManager 구조를 사용하여 충돌 및 아이템 획득 시 사운드 풀 연동 처리

## 직접 개발한 내용

Y축 터치 오프셋 이동 및 회전 로직: 터치한 손가락 위치보다 항상 150f 위에서 기체가 따라오도록 오프셋을 적용하고, 이동 방향에 맞춰 기체가 실시간으로 회전하도록 구현

GameContext 매개변수 직접 전달 방식을 통한 참조 구조 개편: CollisionChecker 및 아이템 클래스 구조를 전면 개편하여 불필요한 멤버 변수 참조를 제거하고 시스템 컴파일 에러(Unresolved reference) 완벽 해결

MainScene 중심의 게임 흐름 제어 및 재시작 시스템: START, PLAY, GAMEOVER 상태를 연결하고, 플레이어 사망 시 화면 터치로 forEachReversedAt을 가동해 필드 위 탄환/적/운석 레이어를 완전 소거한 뒤 스탯과 스코어를 리셋하는 초기화 루프 구현

확률형 아이템 드롭 및 보호막 무적 메커니즘: 적 격추 시 확률적으로 회복(15%) 및 보호막(8%) 아이템이 나오도록 스폰 연동을 하고, 보호막 획득 시 플레이어 내부에 4초 무적 타이머가 가동되어 충돌 데미지를 무효화하는 로직 구현

조준 방향 수직 벡터 기반의 1자 평행 탄막 확장 알고리즘: 점수대별(0점 / 1500점 / 3500점) 무기 티어 분기를 적용하고, 기체의 조준 방향에 완벽히 수직인 좌우 평행 벡터를 계산하여 총알이 꺾이지 않고 정면 1자 대형을 유지하며 발사 개수가 늘어나는(1발 -> 2발 -> 3발) 사격 패턴 구현

사인(sin) 함수 기반의 지그재그 회피 기동 특수 적 구현: 일반 적보다 크기 1.5배, 내구도 3배의 보스급 객체인 '특수 적 드론'을 추가하고, 매 프레임 누적 시간에 사인 함수를 대입해 횡방향으로 흔들리며 내려오는 변칙 패턴 및 처치 시 회복 아이템 100% 드롭 기능 연동

# 9. 프로젝트 후기

## 👍 잘된 점 (성과)
* **모듈화 및 객체 지향적 역할 분담 성공 (구조적 체계성 확보)**
  - `Player`, `Bullet`, `Enemy`, `Asteroid` 등 핵심 엔티티들을 개별 `.kt` 파일과 독립된 클래스로 명확히 분리하여 설계했습니다.
  - 충돌 검사(`CollisionChecker`)와 사운드 제어(`SoundManager`), 아이템 드롭 메커니즘을 각각 전담 함수와 객체로 쪼개어 관리함으로써 코드의 가독성을 높였습니다. 
  - 최종 단계에서 `GameContext` 매개변수 직접 전달 방식으로 구조를 개편할 때도, 이처럼 클래스와 함수가 역할별로 깔끔하게 분류되어 있었던 덕분에 복잡한 리팩토링 작업을 큰 부작용 없이 안정적으로 완수할 수 있었습니다.

## 📐 아쉬운 점 (한계 및 시행착오)
* **시행착오와 일정 관리의 교훈**
  - 2차 발표 시점까지 기존 계획대로 안정적인 빌드를 확보한 후, 최종 마무리를 하면서 더 완성도 높은 게임을 만들고 싶다는 욕심에 복잡한 수동 무기 교체 상점 기능을 무리하게 추가하려고 시도했습니다.
  - 그러나 프레임워크 내부의 레이어 관리 큐와 다수의 컴포넌트 간 참조가 복잡하게 얽히면서 심각한 컴파일 결함이 발생했고, 제한된 마감 시간 내에 구조적 결함을 완전히 해결하기 어렵다고 판단했습니다. 
  - 결국 중간 무기 상점 관련 기능들을 통째로 폐기하고, 대신 점수대별 1자 평행 탄막 사격 시스템과 특수 적 패턴 강화로 선회하여 내실을 다졌습니다. 이 과정을 통해 개발 초기 단계에서 실현 가능한 스코프를 명확히 설정하고 일정에 맞춰 유연하게 기획을 조율하는 능력이 얼마나 중요한지 뼈저리게 깨달았습니다.

## 🎓 배운 점 (역량 성장)
* **모바일 플랫폼 및 모던 언어 프레임워크에 대한 깊이 있는 이해**
  - **모바일 UX 및 UI 제어 경험:** 터치한 자리에 기체가 가려지는 모바일 특유의 물리적 한계를 해결하기 위해 Y축 오프셋을 직접 계산해 매핑하고, 제한된 화면 크기 안에서 적의 다양한 이동 패턴을 연산해 보며 모바일 디바이스 환경에 특화된 UI/UX 설계 감각을 익혔습니다.
  - **Android Studio 프로그래밍 도구 숙련:** 안드로이드 프레임워크를 직접 라이브러리 모듈로 연결하고 빌드 환경(Gradle)을 동기화하는 과정을 통해, 통합 개발 환경(IDE) 내에서 발생하는 결함을 추적하고 디버깅하는 실무 능력을 길렀습니다.
  - **Kotlin 언어 핵심 메커니즘 체득:** 코틀린 언어가 가진 강력한 확장성과 타입 시스템을 다루며, 프레임워크 내부 루프 내에서 메모리 누수 없이 객체를 지우는 `forEachReversedAt` 등의 구조를 직접 다뤄보는 값진 경험을 얻었습니다.
